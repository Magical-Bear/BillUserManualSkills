# AI Architecture Reference

1. Dify 工作流
2. LangChain/Agent
3. RAG
4. 微调和推理

## 1. Dify 工作流

明确输入变量、节点、知识库、工具、流式输出、错误分支、超时、重试、API Key 存放和审计日志。

## 2. LangChain/Agent

先查官方文档。确认模型接口、工具 schema、中间件、状态、回调、超时、重试、人类确认和 trace。

## 3. RAG

定义文档来源、清洗、切分、metadata、embedding、向量库、召回数量、重排、引用格式、权限过滤、更新频率和评估集。

## 4. 微调和推理

微调需要数据授权、脱敏、训练/验证/测试划分、指标、可复现配置和模型卡。推理需要显存预算、并发、上下文长度、批处理、量化、限流和健康检查。

## 5. 多模态链路

典型链路：唤醒词检测 → 录音 → ASR → 拍照/图像输入 → Dify 或 Agent → 流式响应 → TTS。必须说明延迟预算和失败兜底。

## 6. 评估和安全

至少覆盖准确性、引用命中、幻觉、延迟、成本、安全拒答、隐私泄露和工具误用。生产环境保留 prompt/workflow 版本和 trace id。
